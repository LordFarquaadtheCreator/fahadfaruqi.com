Working across stacks is only useful when it improves the product. The goal is not to collect tools, but to know enough of the system to remove friction wherever it appears.

That might mean tuning a SwiftUI flow, simplifying an API, cleaning up deployment, or shaping a prototype quickly enough for the team to learn from it.

```ts
const range = ['mobile', 'web', 'backend', 'ai'].filter((tool) => tool.movesProductForward);
```

Range matters most when it shortens the path between idea and working product.
